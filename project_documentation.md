# Google Apps Script 量化回測系統 - 詳細專案文件

本文件深入解析專案中 15 個程式檔案的內部邏輯、關鍵函式與運作原理。

**最後更新**: 2026-01-10

## 1. 核心入口與設定 (Core)

### [Code.gs](Code.gs)
**角色**：程式入口點 (Entry Point)
**詳細說明**：
這是 Google Apps Script 專案的「大門」。當使用者打開試算表時，Google 伺服器會尋找此檔案中的 `onOpen` 函式。
- **關鍵函式**：
  - `onOpen()`：使用 `SpreadsheetApp.getUi()` 建立自定義選單。這是使用者唯一需要互動的地方。
  - `setupAndFetchData()`：選單「1」的觸發器，直接呼叫 `MarketDataFetcher`。
  - `runBacktest()`：選單「2」的觸發器，直接呼叫 `RebalanceController`。
  - `showAnalysis()`：選單「3」的觸發器，呼叫 `Analysis` 顯示報表。
  - `clearBacktestResults()`：選單「4」的觸發器，清除所有回測結果。
  - `drawPerformanceChart()`：選單「5」的觸發器，繪製績效走勢圖。

### [Constants.gs](Constants.gs)
**角色**：全域設定檔 (Configuration)
**詳細說明**：
集中管理所有「變數」，讓調整策略無需修改程式碼邏輯。
- **重要參數**：
  - `INITIAL_CAPITAL`: 初始資金 (10000 USD)。
  - `TICKERS`: 投資標的 `['VGT', 'VDC', 'XLE', 'GLTR', 'TSLA', 'SGOV']`。
  - `STRATEGY_LAYERS`: 定義 4 層策略邏輯 (危機偵測、鎖倉、趨勢/跳空、動態評分)。
  - `CONSTRAINTS`: 
    - `MA_TREND_DAYS`: 100日 均線濾網。
    - `GAP_UP_MULTIPLIER`: 跳空買入判定 (Open > High * 1.005)。
    - `COST_SKIP_THRESHOLD`: 換手率 (Turnover) ≤ 10% 且無危機時跳過再平衡。
    - `SCORE_BONUS`: 強勢股與跳空加碼獎勵提升至 10%。
  - `COST_CONFIG`: 交易成本 (最低 $1, 每股 $0.0035) 與 再平衡稅費 (0.1%)。

### [Utils.gs](Utils.gs)
**角色**：數學與統計工具箱
**詳細說明**：
純粹的數學運算層，不涉及任何業務邏輯或試算表操作。
- **關鍵函式**：
  - `calculateMA(slice, ticker)`：計算給定區間的算術移動平均線。
  - `calculateSharpe(slice, ticker)`：計算年化夏普比率 (Sharpe Ratio)，用於評分。
  - `calculateCorrelation(slice, t1, t2)`：計算兩資產間的皮爾森相關係數 (Pearson Correlation)，用於風險控制。
  - `calculateBeta(slice, asset, bench)`：計算資產相對於大盤 的 Beta 值，衡量系統性風險。

### [Logger.gs](Logger.gs)
**角色**：日誌封裝
**詳細說明**：
簡單的封裝層，未來若需將日誌寫入試算表或外部資料庫，只需修改此檔案，無需更動其他程式碼。

### [Notifier.gs](Notifier.gs)
**角色**：使用者通知中心
**詳細說明**：
負責與使用者溝通的介面。包含 try-catch 處理以支援非互動模式執行（如時間觸發器）。
- **關鍵函式**：
  - `alert(msg)`：在試算表畫面上跳出彈出視窗 (Modal Dialog)。
  - `sendEmail(subject, body)`：檢查 `Constants` 是否有設定 Email，若有則透過 `MailApp` 寄送 HTML 格式的信件。

## 2. 資料層 (Data Layer)

### [MarketDataFetcher.gs](MarketDataFetcher.gs)
**角色**：資料抓取與清洗 (ETL)
**詳細說明**：
負責將外部數據 (Yahoo Finance) 轉換為系統可用的內部格式。
- **運作流程**：
  1. **建立年度資料表**：不再使用單一「Data」工作表，而是依年度拆分（如 `2010Data`, `2011Data`...）以免列數過多。
  2. 針對每個標的 (Ticker) 呼叫 Yahoo Finance API 下載 CSV。
  3. **資料回填 (Backfill)**：這是關鍵邏輯。若某標的 (如 SGOV) 在回測起始日尚未上市，程式會抓取該標的「上市第一天」的價格，並將其回填至上市前的所有日期。這確保了回測矩陣的完整性，並將缺失數據視為「持有現金」(價格不變)。
  4. 將清洗後的數據 (OHLCV) **按年份寫入**對應的年度工作表 (例如 `2023Data`)。
- **支援標的**：VGT, VDC, XLE, GLTR, TSLA, SGOV, SPY (大盤), ^VIX (恐慌指數)。

### [DailyLogger.gs](DailyLogger.gs)
**角色**：高效能結果記錄器
**詳細說明**：
解決 Google Apps Script 執行時間與記憶體限制的關鍵組件。
- **優化技術**：
  - **批次寫入 (Batch Writing)**：它不會每算一天就寫入一次試算表 (這會極慢)。而是提供 `appendBatch(results)` 函式，接收一個陣列 (例如 200 天的結果)，一次性呼叫 `setValues` 寫入。這大幅減少了與 Google Sheet API 的通訊次數。
- **增量更新支援**：
  - `getLastProcessedDate()`：取得最後處理的日期。
  - `getLastPortfolioState()`：恢復投資組合狀態（持股、淨值）。
  - `deleteLastNRows(n)`：刪除最後 N 筆資料，用於重新處理最近的交易日。
  - `getEndOfYearValues(year)`：取得指定年份的年底價值，用於 YTD 計算。
- **輸出欄位**：
  - 日期、投資組合價值
  - VGT/VDC/XLE/GLTR/TSLA/SGOV/VIX % (各標的當日變動)
  - 當年內大盤績效/此策略績效 (YTD 報酬)
  - 訊號、策略說明
  - **操作** (詳細交易記錄：買入/賣出股數、費用、獲利/虧損)
  - VGT/VDC/XLE/GLTR/TSLA/SGOV 股數/佔比

## 3. 邏輯運算層 (Logic Engines)

### [FilterEngine.gs](FilterEngine.gs)
**角色**：策略濾網 (Crisis, Trend, Gap)
**詳細說明**：
提供無狀態的邏輯判斷函式，供 Controller 呼叫。
- **邏輯細節**：
  - `checkCrisis(vix, mdd)`：第一層危機偵測 (VIX $\ge$ 25 或 5日 MDD $\ge$ 7%)。
  - `checkTrend(prices, ma100)`：第三層過濾，檢查價格是否低於 100 日均線且月報酬為負。
  - `checkGapUp(open, prevHigh)`：第三層跳空補償，偵測開盤價是否大於前日最高價 $\times 1.005$。

### [ScoringEngine.gs]
**角色**：動態評分系統 (Layer 4)
**詳細說明**：
負責計算各資產的強弱勢得分 (滿分 4.0)，用於動態權重調整。
- **評分項目**：近1月報酬、近3月報酬、夏普比率排名、相對 Alpha。
- **強者恆強**：滿分且排名第一者可獲得額外權重加成。

## 4. 執行與管理層 (Execution & Management)

### [PortfolioManager.gs](PortfolioManager.gs)
**角色**：帳戶狀態管理
**詳細說明**：
模擬證券帳戶，支援現金緩衝與全額交割。
- **狀態追蹤**：
  - `cash`：現金餘額。
  - `holdings`：各標的持有「股數」(Units)。
  - `avgCosts`：加權平均成本。
- **功能**：
  - `updateValue`：更新總資產。
  - `getTotalCost`：計算總成本（用於計算未實現損益）。

### [RebalanceExecutor.gs](RebalanceExecutor.gs)
**角色**：交易執行員
**詳細說明**：
負責執行「目標權重」的買賣指令。
- **交易機制**：
  - **整股交易**：使用 `Math.floor` 確保只買賣整數股。
  - **指定價格**：支援 Stop Loss 或 Gap Up 指定成交價 (如 Open 或 Stop Price)。
- **成本模型** 
  - 最低手續費：$1.00 / 筆
  - 股數費率：$0.0035 / 股
  - 再平衡稅費：每次再平衡價值 × 0.1% (模擬滑點與稅)
  - 兩者取大值。

### [RebalanceController.gs](RebalanceController.gs)
**角色**：主控制器 (Strategy Core)
**詳細說明**：
實現 4 層優先級的量化核心策略。
- **核心策略流程** (每日判斷 / 每月再平衡)：
  1. **第一層：危機偵測與防禦配置 (Crisis Mode)**：
     - 若 VIX $\ge$ 25 或 5日 MDD $\ge$ 7%，進入防禦模式。
     - **避險配置**：100% SGOV/Cash (極度保守，完全避開股市波動)。
     - **解除條件**：需滿足連續 2 日 VIX < 20 且 連續 3 個交易日，當日 VIX < 20 且 SPY 當日最低價 >= 前一日最低價 才解除。
     - **不虧損原則**：即使在危機模式，若現有持倉虧損，強制鎖定不賣出 (Override Crisis Weights)。
  2. **第二層：嚴格鎖倉保護 (No Loss Sell)**：
     - 若現價 < 平均成本，強制鎖定該標的目標權重 = 當前權重。
     - 確保不實現虧損 (SGOV 除外，採含息判斷)。
  3. **第三層：趨勢過濾與跳空 (Trend & Gap)**：
     - 若 < 100MA 且月報酬 < 0，降權重至 0 (除非鎖倉)。
     - 若向上跳空 (今日最低價 > 昨日最高價) 且趨勢向上，額外加碼。
     - **跳空類型邏輯條件 (判斷式)**：
       - **真空區域 (Gap Range)**：昨日最高價 $\leftrightarrow$ 今日最低價 之間的價差
       - **向上跳空**：今日最低價 > 昨日最高價 (加碼)
       - **向下跳空**：今日最高價 < 昨日最低價 (不用加碼, 只有向上跳空才加碼) 
  4. **第四層：動態評分 (Dynamic Scoring)**：
     - 依據 1M/3M 報酬、Sharpe、Alpha 計算得分。
      - 強勢股獲得額外權重分配。
   5. **第五層：權重上限控制 (Max Weight Caps)**：
      - VDC, XLE, TSLA 上限 30%。
      - GLTR 上限 10%。
      - **例外**：若「不虧損鎖倉」導致部位已超過上限，則以鎖倉為準 (不賣出)。
   6. **再平衡與成本控制**：
     - **每月再平衡**：預設每月最後交易日執行。
     - **跳過規則**：若換手率 (Turnover) $\le$ 10% 且無危機訊號，沿用上期權重以節省成本。
     - **決策透明化**：在日誌中詳細記錄「未達偏離」、「忽略偏離」或「觸發再平衡」的具體數值原因，以及強勢股得分細節。
     - **強制寄信**：無論是否有交易，回測結束皆發送報告。
- **YTD 計算**：
  - 支援增量更新模式下的 YTD 正確計算。

## 5. 分析與工具層 (Analysis & Tools)

### [RebalanceCalculator.gs](RebalanceCalculator.gs)
**角色**：參數化再平衡試算工具
**詳細說明**：
提供獨立的試算表介面，讓使用者可以快速計算再平衡操作建議。
- **功能特點**：
  - 自動建立「Rebalance Calculator」工作表。
  - 即時抓取 Yahoo Finance 最新價格。
  - 根據使用者輸入的總資金、當前權重、目標權重，自動計算：
    - 當前持股數量
    - 目標持股數量
    - 建議操作 (買入/賣出股數)
  - 自動歸一化目標權重總和。
- **使用方式**：
  1. 首次執行會建立工作表並設定預設參數。
  2. 在 B1 輸入總資金 (USD)。
  3. 在 A4 開始輸入標的、當前權重%、目標權重%。
  4. 再次執行功能，系統會自動抓取價格並計算建議操作。
- **輸出欄位**：
  - 即時現價、現股、目標股、建議操作。

### [Analysis.gs](Analysis.gs)
**角色**：前端報表與視覺化工具
**詳細說明**：
提供使用者直接在 Google Sheet 查看結果的便捷功能。
- **功能**：
  - `showLastRebalances(count)`：
    - 讀取 `Backtest Results` 工作表
    - 過濾出有「訊號」的行 (即發生再平衡或危機觸發的日子)
    - 擷取最後 N 筆記錄
    - 動態生成 HTML 表格，並透過 `SpreadsheetApp.getUi().showModalDialog` 顯示給使用者
  - `drawChart()`：
    - 讀取回測結果數據
    - **智能重建大盤價值**：從 YTD 報酬反推絕對價值序列
      - 追蹤年度變化
      - 使用前一年底的價值作為新年度基準
      - 計算：當前價值 = 年初價值 × (1 + YTD%)
    - 建立「Performance Chart」工作表
    - 寫入日期、策略淨值、大盤淨值數據
    - 生成折線圖比較策略 vs 大盤績效
    - 圖表規格：1000×600，包含標題和軸標籤
